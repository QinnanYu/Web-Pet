🐾JiMaoBao 桌宠项目

一个基于 HTML + CSS + JavaScript 实现的可交互桌宠网页应用。
用户可以与宠物进行多种互动，宠物会根据状态变化、时间变化以及用户行为做出不同反应，实现轻量级“养成 + 桌宠”体验。

✨ 功能概览
🎮 基础功能
可拖拽宠物，在页面自由移动
右键菜单进行互动操作
状态栏显示幸福值与能量值
自定义背景（支持图片）
📊 状态系统
幸福值（Happiness）
能量值（Energy）

特性：

每 10 秒自动下降
不同数值触发不同动画状态：
normal / working
hungry / crying
angry / boring
dead（死亡）
🐶 宠物行为系统
1. 手动互动

通过右键菜单触发：

行为	效果
🥰 贴贴	幸福↑ 能量↓
🏃 锻炼	幸福↑ 能量↓↓
🔋 充电	幸福↑ 能量↑
🍰 投喂	恢复状态 / 吃撑
🐾 遛小鸡毛	幸福↑ 能量↓
✨ 随机出现	瞬移位置
2. 自动行为

宠物会在空闲时自动执行行为：

发呆
营业（working）
随机换位置
自言自语（气泡提示）
3. 自动行走系统（核心特色）
宠物会随机开始“散步”
左右移动
碰到边缘自动掉头
拖拽 / 操作时自动停止

👉 让宠物更“像活着”

💬 对话气泡系统
替代 alert() 弹窗
在宠物旁显示提示信息
自动跟随宠物位置

示例：

“小白，我在发呆哦~”
“今天也要认真营业！”
“我又活过来啦！”
🎁 彩蛋系统
疯狂点击彩蛋
1 秒内点击 5 次触发
播放特殊动画 caidan.gif

显示：

小白，我被rua炸毛啦！！

💤 无聊机制

长时间不互动：

宠物进入 boring 状态

自动提示：

小白，你怎么不理我啦……

⏰ 整点报时
每到整点触发
切换 clock 状态
显示当前时间提示
☠️ 死亡与复活

当幸福值和能量值同时为 0：

宠物进入死亡状态
30 秒后自动复活
恢复部分属性
👁️ 显示控制
显示 / 隐藏宠物
显示 / 隐藏状态栏
